# DataOps Studio 演示指南

## 快速启动

```bash
# 方式一：双击启动
双击 start.bat

# 方式二：手动启动
# 终端1 — 后端
cd backend && pip install -r requirements.txt && python -m uvicorn main:app --reload --port 8000

# 终端2 — 前端
cd frontend && npm install && npx vite
```

启动后访问 http://localhost:5173 看到数据运维平台界面。

---

## 演示核心理念

> **不做 PPT 演讲。上来就操作，让领导亲眼看到 AI 实时改变一个真实系统。**

每个场景控制在 **5 分钟以内**，结构统一：
1. 打开系统 → 30秒介绍 "这是我们的数据运维平台"
2. 打开 Claude Code → 演示者操作一个震撼场景
3. 交给领导 → 领导亲手输入需求，看 AI 执行

---

## 场景 A：公司战略级领导

**关注点**：AI 对研发效率的战略价值、降本增效、竞争力

### 开场 (30秒)

打开浏览器展示 DataOps Studio 运维看板：
> "领导，这是我们数据平台的运维管理系统，管理着 6 条核心数据管道，每天处理千万级数据。"

### 第一击：AI 实时创建 CEO 仪表盘 (2分钟)

打开 Claude Code 终端，输入：

```
给 Dashboard 页面顶部增加一个「CEO 执行摘要」区域：
用一个深蓝色渐变背景的大卡片，包含 4 个核心指标：
1. 本月数据处理总量（从 total_rows_30d 计算，显示为 "x.x 亿条"）
2. 平台健康度（用活跃管道数/总管道数 计算百分比）
3. 数据质量评分（已有 quality_score）
4. 月度计算成本（已有 total_cost_30d，显示为 "¥xx,xxx"）
每个指标用大字体白色数字展示，下方带小字说明
```

**效果**：Claude Code 自动修改 Dashboard.jsx → Vite 热更新 → 浏览器实时出现 CEO 仪表盘

> "从需求描述到上线，传统开发需要排期、设计、开发、测试，一般 1-2 周。AI 刚刚用了不到 1 分钟。"

### 领导亲手操作 (2分钟)

> "领导您来试试。您觉得还想在这个面板上看到什么数据？直接用中文告诉 AI。"

**建议引导词**（如果领导不知道说什么）：
- "加一个数据安全合规指标"
- "加一个同比环比趋势"
- "把成本数字改成美元显示"

### 收尾金句

> "传统方式：需求 → PRD → 排期 → 开发 → 测试 → 上线，2周。AI 方式：描述 → 完成，1分钟。这就是 AI 对研发效率的变革。"

## 场景 B：研发体系技术竞争力领导

**关注点**：代码质量、技术债务、架构能力、AI 是否真的懂代码

### 开场 (30秒)

先展示系统，指出质量页面的红色告警：
> "这个系统里故意留了一些真实项目中常见的问题，看看 AI 能不能发现。"

### 第一击：AI 代码审查发现隐藏 Bug (2分钟)

```
请全面审查这个 DataOps Studio 项目的所有配置文件和代码，找出潜在的 bug、技术债务和安全风险，按严重程度排序
```

**Claude Code 会发现**（都是预埋的真实问题）：
1. `QR-005` 质量规则 enabled=false — 上线时忘记启用，导致脏数据
2. `risk_score_calc` 仍在调用 legacy 脚本，新引擎已就绪但未切换
3. `QR-003` freshness 阈值过严，高峰期必然误报
4. `permission.yaml` 认证未启用 (auth.enabled=false)
5. `user_profile_sync` 的 quality_gate_enabled=false

> "这些都是真实项目中会出现的问题。人工 Code Review 很容易遗漏配置文件里的逻辑问题，AI 可以理解业务语义。"

### 第二击：AI 修复问题 (1分钟)

```
修复 QR-005 的 bug：启用这条质量规则，并且把 user_profile_sync 管道的 quality_gate_enabled 也设为 true
```

**效果**：Claude Code 修改 YAML → 后端 reload → 刷新前端 → 质量页面的禁用告警消失

### 领导亲手操作 (2分钟)

> "您选一个技术债务，让 AI 修复。比如让它把 risk_score_calc 迁移到新引擎。"

```
把 risk_score_calc 管道的配置从 legacy 引擎切换到新引擎：关闭 use_legacy_engine，移除 legacy_script 配置，把状态从 degraded 改为 active
```

### 收尾金句

> "AI 不只是写代码的工具，它能理解业务上下文，发现人很难注意到的配置逻辑问题。对技术竞争力来说，这是质的提升。"

## 场景 C：研发管理侧领导 (HR/质量运营)

**关注点**：团队效能度量、质量运营、人效提升

### 开场 (30秒)

打开成本分析页面，指出底部的「团队效能概览」表格：
> "这是各团队的基础运营数据。但管理侧需要更直观的视角，比如团队排名、效能对比。让 AI 来做。"

### 第一击：AI 创建团队效能看板 (2分钟)

```
在成本分析页面的团队效能概览上方添加一个可视化区域：
1. 一个横向柱状图，展示各团队的质量评分排名（从高到低）
2. 在柱状图上用不同颜色区分：>=90分绿色，80-90黄色，<80红色
3. 图表标题为「团队质量评分排行榜」
```

**效果**：实时出现团队排行榜可视化

> "管理侧最关心的度量和看板，不需要再等开发排期。直接用自然语言描述，AI 立刻产出。"

### 第二击：AI 生成分析报告 (1分钟)

```
基于当前的数据质量规则和检查结果，在质量管理页面底部添加一个「月度质量分析摘要」Card：
- 显示本月质量检查总次数、通过率
- 列出 TOP 3 高风险问题（未启用的规则、频繁违规的规则）
- 给出改进建议
用文字段落格式展示，像一份迷你报告
```

### 领导亲手操作 (2分钟)

> "您平时最想看到什么团队数据？直接告诉 AI。"

**建议引导词**：
- "加一个各团队每周故障次数趋势"
- "标记出成本最高但质量最低的团队"
- "给每个团队加一个综合效能评分"

### 收尾金句

> "以前管理层要数据，需要提需求给开发，排期可能是下个迭代。现在 AI 让管理决策所需的数据触手可及。这也意味着对研发人员的能力要求在发生变化。"

## 注意事项

1. **演示前务必测试**：运行 `start.bat` 确认前后端正常启动，浏览器能看到完整界面
2. **网络环境**：Claude Code 需要能访问 API，确保网络通畅
3. **备用方案**：如果 AI 生成的代码有小错误，不要慌 — 这本身也是演示点："AI 会犯错，但修正速度极快"
4. **热更新依赖**：前端使用 Vite 热更新，修改 `.jsx` 文件后浏览器自动刷新，无需手动操作
5. **配置热重载**：修改 YAML 配置后，访问 `http://localhost:8000/api/config/reload` 即可重载，无需重启后端

## 项目结构

```
dataops-studio/
├── backend/
│   ├── configs/
│   │   ├── pipelines.yaml      ← 管道配置（内含预埋 bug）
│   │   ├── quality.yaml        ← 质量规则（内含禁用规则 bug）
│   │   └── permission.yaml     ← 权限配置（未启用认证）
│   ├── core/                   ← 核心模块（占位）
│   └── main.py                 ← FastAPI 后端 API
├── frontend/
│   └── src/
│       ├── components/Layout.jsx   ← 全局布局
│       └── pages/
│           ├── Dashboard.jsx       ← 运维看板（场景A主战场）
│           ├── Pipelines.jsx       ← 管道管理（场景B涉及）
│           ├── QualityRules.jsx    ← 质量管理（场景B/C涉及）
│           └── CostAnalysis.jsx    ← 成本分析（场景C主战场）
├── start.bat                   ← 一键启动
└── DEMO_GUIDE.md               ← 本文件
```
