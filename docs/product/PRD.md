# DataOps Studio 产品需求文档

**版本**: v1.0
**日期**: 2026-01-31
**作者**: Jianmin Lu
**状态**: Active

## 1. 产品定位

DataOps Studio 是公司内部数据平台的统一运维管理系统，面向数据工程师、数据团队管理者和公司管理层，提供数据管道全生命周期管理能力。

## 2. 目标用户

| 用户角色 | 核心诉求 |
|---------|---------|
| 数据工程师 | 管道运维、问题排查、配置管理 |
| 数据团队 Leader | 团队效能、质量趋势、成本控制 |
| 公司管理层 | 平台健康度、资源投入 ROI、风险感知 |

## 3. 功能模块

### 3.1 运维看板 (Dashboard)

- 核心指标卡片: 活跃管道数、执行次数、质量评分、成本总览
- 执行趋势图 (14 天)
- 管道状态分布
- 最近告警列表

### 3.2 数据管道管理 (Pipelines)

- 管道列表: 状态、调度频率、成功率、耗时、成本
- 管道详情: 配置信息、源表/目标表、依赖关系
- 执行历史: 时间线视图
- 状态筛选与排序

### 3.3 数据质量管理 (Quality)

- 质量规则列表: 启用状态、严重级别、检查类型
- 规则启用/禁用状态检测与告警
- 质量评分趋势 (14 天)
- 检查结果明细

### 3.4 成本分析 (Cost)

- 成本概览: 30 天总成本、日均成本、环比变化
- 每日成本趋势
- 管道成本占比 (饼图)
- 管道成本明细表
- 团队效能概览: 成功率、质量评分、数据处理量

## 4. MVP 范围

### 包含

- 上述 4 个功能模块完整 UI
- YAML 驱动的配置管理
- 模拟数据生成 (30 天历史)
- 配置热重载
- 数据血缘 API

### 不包含 (后续迭代)

- 用户认证与权限控制
- 管道调度执行引擎
- 数据库持久化存储
- 告警通知 (邮件/IM)
- 数据血缘可视化页面

## 5. 非功能需求

- 前端首屏加载 < 2 秒
- 后端 API 响应 < 200ms
- 支持 Chrome/Edge 最新两个大版本
- 前端热更新, 修改即可见
